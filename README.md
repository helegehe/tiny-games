### 1. 游戏概述

#### 1.1 一句话简介

《别点我》是一款考验玩家逆向思维与快速反应的休闲小游戏，玩家需根据指令点击“不符合条件”的图案，在有限时间内做出正确选择。

#### 1.2 目标用户

- **核心人群：** 16-30岁学生及职场新人，喜欢挑战反应速度、热衷分享搞笑内容。
- **次核心人群：** 30-45岁轻度玩家，利用碎片时间放松，容易被简单规则吸引。
- **潜在人群：** 家庭亲子用户，父母带孩子一起玩，培养观察力。

#### 1.3 游戏定位

- **类型：** 超休闲+脑力训练
- **平台：** 微信小游戏
- **核心体验：** 魔性、紧张、成就感、社交话题

### 2. 核心玩法详解

#### 2.1 基本规则

- 每轮游戏屏幕上方显示一条**指令**（如“不要点水果”）。
- 下方显示**4-6个图案**（图标+文字标签），每个图案具有多重属性（类别、颜色、形状等）。
- 玩家需在**倒计时内**点击**不符合指令条件的图案**。
- 点击正确进入下一轮，点击错误或超时则累积一次失误。
- 累计失误3次，游戏结束。

#### 2.2 指令与图案的匹配逻辑

每个图案由以下属性构成：

- **类别：** 水果、动物、交通工具、文具、食物、日用品等
- **颜色：** 红、黄、蓝、绿、紫、黑、白等
- **形状：** 圆形、方形、三角形、不规则等
- **大小：** 大、中、小（特殊关卡使用）
- **数量：** 1个、2个、多个（用于数字指令）

指令类型分为三级难度：

- **L1-单属性禁止：** “不要点水果”（类别）、“不要点红色”（颜色）、“不要点圆形”（形状）
- **L2-复合属性禁止：** “不要点红色的水果”（颜色+类别）、“不要点会飞的动物”（动作+类别）
- **L3-逻辑否定：** “点不是水果的”、“除了红色都可以点”（反向表述）
- **L4-比较判断：** “不要点比3大的数字”、“点最小的动物”

#### 2.3 操作与反馈

- **点击区域：** 每个图案占据独立区域，支持手指触摸。
- **视觉反馈：**
  - 点击正确：图案炸裂成彩色粒子，伴随清脆音效，下一轮出现。
  - 点击错误：图案变红并抖动，屏幕闪红，伴随“噗”声，失误计数+1。
  - 超时：倒计时归零后，所有图案闪烁，提示超时，失误计数+1。
- **倒计时表现：** 屏幕上方进度条逐渐缩短，最后3秒有滴答声加速。

#### 2.4 特殊机制

- **闪烁干扰：** 部分关卡图案快速闪烁（每秒2-3次），考验眼力。
- **位置互换：** 每0.5秒图案位置随机交换，增加点击难度。
- **真假混淆：** 出现相似图案（如红色苹果和红色草莓），需仔细区分。

### 3. 游戏模式设计

#### 3.1 经典闯关模式（主线）

- **结构：** 每10轮为一关，每关最后1轮为Boss指令（复合条件+时间缩短）。
- **主题包：**
  - 初始包：日常物品（水果、动物、文具）
  - 解锁包：Emoji表情包、数字字母包、电影角色包、网络热梗包
- **进度保存：** 自动记录最高关卡，每日首次通关额外奖励金币。

#### 3.2 限时挑战模式（变现核心）

- **规则：** 60秒内尽可能答对更多轮次，每轮时间固定1秒，答对+1分，答错不扣分但结束（类似“无尽模式”）。
- **复活：** 失误后可看广告立即复活，继续计时。
- **排行榜：** 按当日最高分排名，前100名获得称号奖励。

#### 3.3 双人对战模式（裂变核心）

- **匹配机制：** 邀请微信好友或随机匹配。
- **玩法：** 双方看到相同指令和图案，同时作答。答对得1分，答错对方得1分。率先达到5分获胜。
- **观战：** 好友可旁观，弹幕互动。
- **奖励：** 胜方获得对方少量金币，每日首胜有额外奖励。

#### 3.4 每日挑战（留存核心）

- **规则：** 每天一个独特主题，如“周二动物夜”“周末数字迷阵”，所有玩家挑战同一组固定种子关卡。
- **奖励：** 完成可获得限定头像框、称号或大量金币。

### 4. 关卡与难度曲线

#### 4.1 初始关卡（1-10关）——教学与适应

| 关卡 | 指令示例        | 图案                                | 时间  | 说明               |
| :--- | :-------------- | :---------------------------------- | :---- | :----------------- |
| 1    | 不要点水果      | [苹果] [汽车] [香蕉] [猫]           | 3秒   | 简单类别区分       |
| 2    | 不要点动物      | [狗] [桌子] [鸟] [书包]             | 3秒   | 同上               |
| 3    | 不要点红色      | [红苹果] [蓝汽车] [红草莓] [黄香蕉] | 2.5秒 | 引入颜色属性       |
| 4    | 不要点圆形      | [足球] [方盒] [月亮] [书]           | 2.5秒 | 引入形状           |
| 5    | 不要点水果+红色 | [红苹果] [蓝莓] [草莓] [黄梨]       | 2秒   | 复合属性（先简单） |
| ...  | ...             | ...                                 | ...   | ...                |

#### 4.2 进阶关卡（11-30关）——复合与干扰

- 引入“不要点红色的水果”（需同时满足两个条件）
- 引入闪烁干扰（图案每秒闪2次）
- 引入位置互换（每0.5秒换位）

#### 4.3 高手关卡（31-50关）——逻辑否定与比较

- “点不是水果的”（反向表述）
- “不要点比3大的数字”（数字比较）
- 真假图案（同时出现红色苹果和红色樱桃，需区分水果具体种类）

#### 4.4 无尽模式难度曲线

- 初始1.5秒/轮，每答对10轮时间减少0.1秒，最低0.8秒。
- 每10轮切换一次主题包，增加新鲜感。

### 5. 美术与音效风格

#### 5.1 美术风格：孟菲斯+涂鸦风

- **色彩：** 高饱和度撞色（粉+蓝、橙+紫），背景用柔和渐变色。
- **图标：** 粗线条、圆润造型，每个图标带简单表情（例如苹果微笑，汽车吐舌），增强亲和力。
- **字体：** 手写粗体，带轻微抖动效果，用于指令文字。
- **UI：** 圆角大按钮，毛玻璃质感，轻拟物。

#### 5.2 动效设计

- **进入游戏：** 图标从屏幕外弹跳进入，伴随“咻”声。
- **点击正确：** 图标爆炸成彩色圆点，散开消失。
- **点击错误：** 屏幕红色闪光，图标抖动并流出眼泪。
- **倒计时：** 周围光圈逐渐收缩，最后1秒闪烁红色。
- **通关：** 烟花绽放，掉落金币。

#### 5.3 音效设计

- **背景音乐：** 轻快8-bit电子乐，节奏随倒计时加快。
- **点击正确：** 清脆的“叮”或“啵”。
- **点击错误：** 搞笑的“噗”或“啊哦”。
- **倒计时：** 滴答声，最后3秒加速。
- **失误提示：** 低沉的“嗡”声。
- **结算：** 欢快的胜利音效。

### 6. 变现系统

#### 6.1 金币体系

- **用途：** 购买主题包、复活、兑换提示、装扮头像框。
- **获取：** 每通关1关获得10金币，每日签到、看广告、好友赠送。

#### 6.2 激励视频广告场景

| 场景                                   | 用户动机         | 广告收益预估 |
| :------------------------------------- | :--------------- | :----------- |
| 失误后立即复活（继续当前挑战）         | 不想失败，高价值 | 高           |
| 限时挑战中复活并继续计时               | 冲击高分         | 高           |
| 每日双倍金币卡（观看后下一关金币翻倍） | 想快速积累金币   | 中           |
| 解锁新主题包（免费用户需看广告）       | 想体验新内容     | 中           |
| 获得一次提示（自动排除一个错误答案）   | 遇到难题         | 中           |
| 双人对战后翻倍奖励                     | 赢了还想多拿     | 中           |

#### 6.3 插屏广告

- **触发时机：**
  - 每完成3个关卡后，进入下一关前。
  - 游戏结束，查看结算页面时。
  - 每日首次打开游戏后5秒（可跳过）。
- **频率控制：** 同一用户每60秒最多展示1次插屏。

#### 6.4 Banner广告

- **位置：** 游戏界面底部，高度80px。
- **策略：** 仅在主界面和关卡选择界面显示，游戏进行时隐藏，避免干扰。

#### 6.5 原生广告（品牌合作）

- **定制主题包：** 与品牌合作推出专属包（如“可口可乐·不要点可乐”），品牌方付费植入。
- **品牌图标：** 在普通关卡中随机出现品牌图标（如麦当劳薯条），点击后跳转品牌小程序（需用户主动点击）。

### 7. 社交与裂变机制

#### 7.1 好友对战

- **邀请方式：** 点击“邀请好友”分享小程序卡片，好友点击直接进入对战房间。
- **异步对战：** 若好友不在线，可发起“留言挑战”，好友下次上线时收到通知并应战，系统自动结算。

#### 7.2 趣味战绩分享

- **场景：** 每局结束后，生成“大脑状态图”，包含：
  - 本轮正确率（如“正确率90%”）
  - 耗时/得分
  - 搞笑标签（如“脑速超光速”“今天忘带脑子”）
- **分享语：** “你敢挑战我的记录吗？” + 小程序码。

#### 7.3 表情包生成

- 玩家可将自己遇到的奇葩指令+点击结果生成搞笑表情包（如：指令“不要点苹果”，结果点了苹果，生成配文“我就是管不住这手啊”）。
- 支持保存到本地或直接分享给微信好友。

#### 7.4 收集图鉴

- 每遇到一个新图案，自动收录进图鉴。
- 图鉴分为“水果图鉴”“动物图鉴”等，收集满一套可获得称号（如“动物学家”）。
- 缺的图案可通过好友赠送（需要好友拥有该图案，且好友赠送后自己图案不消失）。

#### 7.5 排行榜

- **好友榜：** 显示微信好友的最高关卡/限时挑战分数。
- **全国榜：** 按日/周/总排名，前列玩家获得特殊称号。

### 8. 留存与运营策略

#### 8.1 每日签到

- 连续签到奖励递增：第1天50金币，第2天100金币，第3天150金币+随机称号，第7天稀有皮肤。

#### 8.2 每日任务

- 任务示例：
  - 完成3关经典模式（奖励50金币）
  - 玩一次限时挑战（奖励30金币）
  - 分享一次游戏（奖励20金币）
  - 看一次广告（奖励10金币）

#### 8.3 活动日历

- **周末双倍金币：** 周六日所有关卡金币翻倍。
- **节日限定主题：** 春节期间上线“红包”“鞭炮”主题包，中秋节上线“月饼”“月亮”主题。
- **限时挑战赛：** 每月一次，排名前100获得实物奖励（如定制周边）。

#### 8.4 流失召回

- **推送模板消息：** 用户超过3天未登录，发送“新主题上线，快来挑战”模板消息（需用户授权）。
- **回归奖励：** 回归用户连续3天登录可领取大量金币和稀有称号。

### 9. 技术实现要点

#### 9.1 前端引擎

- **推荐引擎：** Cocos Creator 3.x 或 LayaAir 2.0（微信小游戏支持度高）。
- **关键模块：**
  - 倒计时控制器（基于帧刷新，保证准确）
  - 图案渲染与触摸检测（使用预制体对象池，减少性能开销）
  - 动效系统（使用引擎自带的粒子系统或序列帧动画）

#### 9.2 后端服务

- **推荐方案：** 微信云开发（云数据库+云函数），免运维。
- **数据存储：**
  - 用户信息表：openid、昵称、头像、金币、当前关卡、图鉴收集情况。
  - 排行榜表：每日/每周更新，使用云函数定时聚合。
  - 对战记录表：存储异步对战数据。

#### 9.3 指令

**指令生成逻辑（简化版）：**

- 采用“模板+随机填充”方式，避免硬编码所有关卡。
- 例：模板“不要点[颜色]的[类别]” → 从颜色库随机取一个颜色，从类别库随机取一个类别，然后筛选图案库中符合条件的图案作为陷阱，其他作为正确答案。

#### 9.4 性能优化

- 图案图片采用精灵表（Sprite Sheet），减少Draw Call。
- 使用对象池复用图案节点，避免频繁创建销毁。
- 限制同屏特效数量，老旧机型自动降低特效等级。

### 10. 数据指标体系

#### 10.1 核心指标

- **DAU/MAU**（日活/月活）
- **留存率**（次日、7日、30日）
- **平均时长**（人均游戏分钟数）
- **广告变现相关**：广告展示次数、广告点击率、eCPM、ARPU
- **关卡通过率**（各关卡平均失误次数，用于调整难度）

#### 10.2 用户行为漏斗

- 新用户进入 → 完成第1关（流失率）
- 完成第3关 → 进入限时挑战（转化率）
- 点击分享 → 成功拉新（K因子）

#### 10.3 版本迭代依据

- 各主题包的解锁率、使用时长
- 各广告位的点击率与填充率
- 用户反馈渠道（微信客服、评价）

| 风险                   | 可能性 | 影响 | 应对措施                                                     |

  - 首页提供“声音开关”按钮

- `tools/gen_assets.js`
- 可重复执行：`node tools/gen_assets.js`


### 图片渲染接入状态

已新增资源加载器：`src/assets.js`，并接入到 `src/main.js` 的绘制流程：

- 背景：优先 `assets/images/title_bg.svg`
- 按钮：优先 `assets/images/button_primary.svg`
- 金币图标：优先 `assets/images/icon_coin.svg`
- 卡片底图：优先 `assets/images/card_placeholder.svg`

#### 回退策略

若某资源加载失败（例如部分微信环境对 SVG 支持受限），会自动回退到 Canvas 原生绘制，不影响游戏正常运行。
